		RESTful - 
REST는 개념입니다.
정확한 표현은 RESTful API가 맞습니다.
REST한 개념을 HTTP API에 도입한 것이 RESTful API인거죠.

REST 개념을 도입하지 않고 API 만들어 쓸 수 있습니다.
REST 개념이 퍼지기 전까진 각자 자기만의 규칙을 만들어서 HTTP API를 개발했죠.
REST가 나오기 전에는 HTTP를 응용한 API가 존재하지 않았던게 아닙니다. 
조금 다른 방식과 포맷으로 이용했을 뿐이죠.

RESTful API를 개발하는지를 보시려면 네이버나 카카오, Google 같은 공룡 밴더사의 Open API를 참고하시면 됩니다.

REST의 개념을 아는게 가장 중요하고, 그 다음으로 인증이나 권한, rate limit 기능을 공부해 나가는게 순서입니다.

유지보수 하시면서 브라우저에서 Ajax로 WAS와 데이터 통신을 해본 경험이 있으실걸로 생각됩니다.
이게 HTTP API입니다.
여기에 REST란 개념을 추가하면 RESTful API가 되는거죠.

많은 분들이 RESTful API를 만드신다고 REST 개념보다는 어떤 언어나,  프레임워크를 써야하는지를 더 고민하시는거 같아서 말씀드렸습니다.

RESTful API라는건 새로운 기술도 아니고, 특정 프레임워크를 써야만 구현가능한 것도 아닙니다.
그냥 개념입니다. 

URI와 HTTP 메소드만 봐도 뭐하는 API인지 알게하자
앤드포인트를 구분하는 최소단위는 자원(resource)이다.
이 두가지만 알고가도 됩니다.

GET '/ajax/board/getUserBoards?user_no=1'  // HTTP API
GET '/user/1/boards'   // RESTful API
